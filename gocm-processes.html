Delhi, India (GUR) — Processes

Slack Channels & Email Distro

  • OTP requests via designated Slack channels (OTP, UHR/PP/LHR).
  • Leadership channel for escalations and direction.
  • Use site mailbox for non‑real‑time requests and audit trails.

Adhoc

  • Coaching, training, OJT, ship level, team meetings, Vaiden.
  • Submission timeline: raise 5–10 minutes before start time; extensions/cancellations per SOP.
  • Leader approved adhoc follows capacity rules (Code Amber/Blue intervals).

System Issue Self‑Serve

  • Initiate via system issue workflow; leadership mirrors if CCP is unable to self‑serve.
  • Corrections follow ATS 7‑day rule and edit windows.

OTP Deletion / Removal

  • Pre‑planned OTP (COA, LQT, VAI, MEF, SUP) removed during Code Blue decision windows.
  • Real‑time OTP deletion applicable for slots per interval capacity.

Schedule Update

  • Break adjustments: breaks/lunch are not moved using Adhoc; use proper schedule update workflow.
  • Skill/ship changes follow standard forms and leadership approval.
  • SDL override required for absences after 72 hours.

Trade a Shift

  • Leaders document approval via email; ensure within processable limits and capacity per interval.
  • Trade request submitted same day; date‑based requests outside window go to planning.

Lost Hour Recovery

  • One LHR code only will be added for being clocked out.
  • Daily OT/LHR cap: up to 1.5 hours when alongside a shift.
  • Weekly overtime cap: up to 10.5 hours.
  • Cannot code 7 consecutive working days; ensure a rest day exists.
  • India LHR window: 17:30 – 09:30 (+1 day) IST.

Power Pay

  • Follow break patterns per shift length and PP rules (e.g., 6‑hr: 2:15, 4:00, 5:15).
  • Use paid and unpaid break allocations exactly as defined.

Leave

  • Certified PTO, Sick, Director‑approved leaves; raise within 3 days from shift date.
  • Self‑serve when enabled; otherwise leadership mirrors.

Manila, Philippines (MNL) — Guidelines & Reminders

Slack Channels & Email Distro

  • OTP requests — site Slack workflows and real‑time status.
  • Casework / Supervisor Time — raise in correct channels; leadership mirrors when needed.
  • Leadership channel — escalations and guidance.

ADHOCS

  • Coaching, training, ship level, team meetings, OJT, Vaiden.
  • Submission: 5–10 minutes before schedule start time.
  • SDL override — Code Amber/Blue intervals cap CCPs per interval.

Casework L7

  • Effective Oct 2024: CCPs not to submit if not admin or customer follow‑up request.
  • Only leaders and VOs coach; coaching scheduled when required.
  • Late request will not be accommodated.

Supervisor Time (CFST)

  • Submission: must be raised prior to start time.
  • Coding: within shift start only; late requests not accommodated.
  • Self‑serve where Slack workflows are enabled.

System Issue

  • Self‑serve: CCPs are expected to submit as soon as possible and mirror where enabled.
  • Corrections/backtracking: 3‑day rule for editing; leadership mirrors if CCP cannot.

PTO Guidelines

  • Transfer/swap on the same team; shift difference should be the same.
  • Receiving CCP should have PTO balance; credits applied to transferred PTO.
  • Cancellation windows follow local SOP.

Special Leaves

  • Medical LOA — with SDL / Director approval; coding requirements.
  • Pregnancy, maternity/paternity, bereavement — submit per SOP timelines.

Schedule Update

  • Break adjustments; corrections/backtracking; use Slack WF.
  • OTP deletion — pre‑planned and real‑time rules; capacity based.

OTP Deletion

  • Pre‑planned OTP removed during code blue decisions.
  • Real‑time OTP removal adheres to interval capacity caps.
